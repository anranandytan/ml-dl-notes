# Hidden Markov Model (HMM)

> **Keywords:** hidden Markov model, forward-backward algorithm, Baum-Welch, Viterbi, filtering, smoothing, prediction, EM algorithm

---

## 1. Motivation & Intuition

Many real-world processes generate **sequential observations** whose underlying cause is hidden. For example:

- In **speech recognition**, the acoustic signal (observable) is generated by a sequence of phonemes (hidden).
- In **part-of-speech tagging**, the words in a sentence (observable) carry latent grammatical roles (hidden).
- In **bioinformatics**, a DNA sequence (observable) may be produced by different genomic regions (hidden states such as coding vs. non-coding).

A Hidden Markov Model captures this structure with two layers:

1. A **latent Markov chain** $\{z_t\}$ evolves over time according to transition probabilities. The current state depends only on the previous state — the **Markov assumption**.
2. At each time step, the latent state **emits** an observation according to an **emission distribution** that depends only on the current state — the **output independence assumption**.

The elegance of HMMs is that despite the exponential number of possible state sequences ($N^T$ for $T$ steps and $N$ states), all three fundamental inference problems — evaluation, learning, decoding — can be solved in **polynomial time** via dynamic programming.

| Year | Event |
|------|-------|

## 3. Model Definition

An HMM $\lambda = (\pi, A, B)$ is defined by three sets of parameters:

| Symbol | Name | Definition |
|--------|------|-----------|
| $\pi$ | Initial distribution | $\pi_i = P(z_1 = q_i)$, probability of starting in state $q_i$ |
| $A$ | Transition matrix | $A_{ij} = a_{ij} = P(z_{t+1}=q_j \mid z_t=q_i)$, $\sum_j a_{ij}=1$ |
| $B$ | Emission matrix | $B_{ij} = b_i(o_j) = P(x_t=o_j \mid z_t=q_i)$, $\sum_j b_i(o_j)=1$ |

There are $N$ hidden states $\{q_1,\ldots,q_N\}$ and $M$ possible observations $\{o_1,\ldots,o_M\}$.

### 3.1 Conditional Independence Assumptions

**Markov assumption:** The hidden state at time $t+1$ depends only on the state at time $t$, not on earlier states or observations:
$$P(z_{t+1} \mid z_t, z_{t-1}, \ldots, z_1, x_t, x_{t-1}, \ldots, x_1) = P(z_{t+1} \mid z_t)$$

**Output independence:** Each observation depends only on the current hidden state:
$$P(x_t \mid z_t, z_{t-1}, \ldots, z_1, x_{t-1}, \ldots, x_1) = P(x_t \mid z_t)$$

These two assumptions together define the graphical structure of the HMM.

### 3.2 Joint Distribution

Under these assumptions, the joint probability of a state sequence $I = (z_1,\ldots,z_T)$ and observation sequence $O = (x_1,\ldots,x_T)$ factorises as:

$$P(I|\lambda) = \pi_{z_1}\prod_{t=2}^{T} a_{z_{t-1},z_t}$$

$$P(O|I,\lambda) = \prod_{t=1}^{T} b_{z_t}(x_t)$$

$$P(O,I|\lambda) = \pi_{z_1}\prod_{t=2}^{T}a_{z_{t-1},z_t}\prod_{t=1}^{T}b_{z_t}(x_t)$$

---

## 4. Three Fundamental Problems

| Problem | Task | Algorithm | Complexity |
|---------|------|-----------|-----------|
| **Evaluation** | Given $\lambda$, compute $P(O|\lambda)$ | Forward (or Backward) | $O(N^2 T)$ |
| **Learning** | Given $O$, find $\lambda^* = \arg\max P(O|\lambda)$ | Baum-Welch (EM) | $O(N^2 T)$ per iteration |
| **Decoding** | Given $O$ and $\lambda$, find $\hat{I} = \arg\max_I P(I|O,\lambda)$ | Viterbi | $O(N^2 T)$ |

---

## 5. Evaluation: Forward-Backward Algorithm

### 5.1 Naive Computation and Its Cost

The marginal likelihood is:

$$P(O|\lambda) = \sum_I P(O,I|\lambda) = \sum_{z_1}\sum_{z_2}\cdots\sum_{z_T}\;\pi_{z_1}\prod_{t=2}^{T}a_{z_{t-1},z_t}\prod_{t=1}^{T}b_{z_t}(x_t)$$

Direct summation over all $N^T$ state sequences is exponential in $T$. Dynamic programming (memoising shared sub-computations) reduces this to $O(N^2 T)$.

### 5.2 Forward Algorithm

**Definition.** The forward variable $\alpha_t(i)$ is the joint probability of the observations up to time $t$ and being in state $q_i$ at time $t$:

$$\alpha_t(i) = P(x_1, x_2, \ldots, x_t,\; z_t = q_i \mid \lambda)$$

**Initialisation** ($t=1$):
$$\alpha_1(i) = \pi_i\, b_i(x_1), \qquad i = 1,\ldots,N$$

**Recursion** (from $t$ to $t+1$): We derive the recursion step by step.

$$\alpha_{t+1}(j) = P(x_1,\ldots,x_t, x_{t+1},\; z_{t+1}=q_j \mid \lambda)$$

Marginalise over the state at time $t$:

$$= \sum_{i=1}^{N} P(x_1,\ldots,x_t,x_{t+1},\; z_{t+1}=q_j,\; z_t=q_i \mid \lambda)$$

Factor out $x_{t+1}$ using the chain rule:

$$= \sum_{i=1}^{N} P(x_{t+1} \mid x_1,\ldots,x_t,\; z_{t+1}=q_j,\; z_t=q_i,\;\lambda)\cdot P(x_1,\ldots,x_t,\; z_{t+1}=q_j,\; z_t=q_i \mid \lambda)$$

Apply **output independence** — given $z_{t+1}=q_j$, $x_{t+1}$ is independent of all past observations and states:

$$P(x_{t+1} \mid x_1,\ldots,x_t,\; z_{t+1}=q_j,\; z_t=q_i,\;\lambda) = b_j(x_{t+1})$$

Factor the remaining joint using the **Markov assumption**:

$$P(x_1,\ldots,x_t,\; z_{t+1}=q_j,\; z_t=q_i \mid \lambda) = P(z_{t+1}=q_j \mid z_t=q_i,\;\lambda)\cdot P(x_1,\ldots,x_t,\; z_t=q_i \mid \lambda) = a_{ij}\,\alpha_t(i)$$

Combining:

$$\boxed{\alpha_{t+1}(j) = b_j(x_{t+1})\sum_{i=1}^{N} a_{ij}\,\alpha_t(i)}$$

**Termination:**

$$P(O|\lambda) = \sum_{i=1}^{N}\alpha_T(i)$$

This sums the forward variable at the final time step over all possible terminal states.

### 5.3 Backward Algorithm

**Definition.** The backward variable $\beta_t(i)$ is the conditional probability of the future observations given the current state:

$$\beta_t(i) = P(x_{t+1}, x_{t+2}, \ldots, x_T \mid z_t = q_i,\; \lambda)$$

**Initialisation** ($t=T$):
$$\beta_T(i) = 1, \qquad i = 1,\ldots,N$$

(Vacuous: given $z_T$, the probability of observing nothing in the future is 1.)

**Recursion** (from $t+1$ back to $t$):

$$\beta_t(i) = P(x_{t+1},\ldots,x_T \mid z_t=q_i,\;\lambda)$$

Marginalise over the next state $z_{t+1}$:

$$= \sum_{j=1}^{N} P(x_{t+1},\ldots,x_T,\; z_{t+1}=q_j \mid z_t=q_i,\;\lambda)$$

$$= \sum_{j=1}^{N} P(x_{t+1},\ldots,x_T \mid z_{t+1}=q_j,\; z_t=q_i,\;\lambda)\cdot P(z_{t+1}=q_j \mid z_t=q_i,\;\lambda)$$

By the Markov property, future observations are independent of $z_t$ given $z_{t+1}$. Factor $x_{t+1}$ out by output independence:

$$= \sum_{j=1}^{N} P(x_{t+1} \mid z_{t+1}=q_j)\cdot P(x_{t+2},\ldots,x_T \mid z_{t+1}=q_j)\cdot a_{ij}$$

$$\boxed{\beta_t(i) = \sum_{j=1}^{N} a_{ij}\,b_j(x_{t+1})\,\beta_{t+1}(j)}$$

**Termination** via backward variables:

$$P(O|\lambda) = \sum_{i=1}^{N} \pi_i\, b_i(x_1)\,\beta_1(i)$$

This follows because $P(O|\lambda) = \sum_i P(x_1,\ldots,x_T, z_1=q_i) = \sum_i P(x_1|z_1=q_i)\,P(x_2,\ldots,x_T|z_1=q_i)\,\pi_i = \sum_i b_i(x_1)\,\beta_1(i)\,\pi_i$.

### 5.4 Marginals via Forward-Backward

The forward and backward variables combine to give marginal and pairwise posteriors:

**State marginal** at time $t$:
$$P(z_t=q_i \mid O,\lambda) =: \gamma_t(i) = \frac{\alpha_t(i)\,\beta_t(i)}{P(O|\lambda)} = \frac{\alpha_t(i)\,\beta_t(i)}{\sum_{j=1}^N \alpha_t(j)\,\beta_t(j)}$$

**Pairwise marginal** (needed for Baum-Welch):
$$P(z_t=q_i, z_{t+1}=q_j \mid O,\lambda) =: \xi_t(i,j) = \frac{\alpha_t(i)\,a_{ij}\,b_j(x_{t+1})\,\beta_{t+1}(j)}{P(O|\lambda)}$$

## 6. Learning: Baum-Welch Algorithm (EM for HMMs)

### 6.1 EM Setup

We want $\hat{\lambda} = \arg\max_\lambda P(O|\lambda)$. Treating the state sequence $I$ as the latent variable, the EM Q-function is:

$$Q(\lambda, \lambda^{(t)}) = \sum_I \log P(O,I|\lambda)\cdot P(I \mid O,\;\lambda^{(t)})$$

Expanding $\log P(O,I|\lambda) = \log\pi_{z_1} + \sum_{t=2}^{T}\log a_{z_{t-1},z_t} + \sum_{t=1}^{T}\log b_{z_t}(x_t)$:

$$Q(\lambda,\lambda^{(t)}) = \sum_I\!\left[\log\pi_{z_1} + \sum_{t=2}^{T}\log a_{z_{t-1},z_t} + \sum_{t=1}^{T}\log b_{z_t}(x_t)\right] P(I\mid O,\lambda^{(t)})$$

Because each term involves different parameters ($\pi$, $A$, $B$), the M-step separates into three independent maximisations.

### 6.2 M-Step: Updating $\pi$

Collect terms involving $\pi$ from $Q$:

$$\max_\pi \sum_{i=1}^{N} \log\pi_i\cdot P(z_1=q_i \mid O,\lambda^{(t)}) \quad\text{subject to}\quad \sum_{i=1}^N \pi_i = 1$$

Using a Lagrange multiplier $\eta$ for the constraint $\sum_i \pi_i = 1$:

$$\frac{\partial}{\partial\pi_i}\left[\sum_i \log\pi_i\cdot\gamma_1(i) - \eta\left(\sum_i\pi_i - 1\right)\right] = \frac{\gamma_1(i)}{\pi_i} - \eta = 0$$

$$\Rightarrow\quad \pi_i = \frac{\gamma_1(i)}{\eta}$$

Summing over $i$ and using $\sum_i\gamma_1(i)=1$: $\eta = 1$. Therefore:

$$\boxed{\pi_i^{(t+1)} = \gamma_1(i) = \frac{\alpha_1(i)\,\beta_1(i)}{P(O|\lambda^{(t)})}}$$

This is simply the posterior probability of starting in state $q_i$, given the observations and current parameters.

### 6.3 M-Step: Updating $A$

Collect terms involving $a_{ij}$:

$$\max_A \sum_{i=1}^{N}\sum_{j=1}^{N}\left(\sum_{t=1}^{T-1}\xi_t(i,j)\right)\log a_{ij} \quad\text{subject to}\quad \sum_j a_{ij}=1\;\forall i$$

By identical Lagrange multiplier argument:

$$\boxed{a_{ij}^{(t+1)} = \frac{\sum_{t=1}^{T-1}\xi_t(i,j)}{\sum_{t=1}^{T-1}\gamma_t(i)}}$$

**Interpretation:** Expected number of transitions from $q_i$ to $q_j$, divided by expected number of transitions out of $q_i$.

### 6.4 M-Step: Updating $B$

Collect terms involving $b_i(o_k)$ (for discrete observations):

$$\boxed{b_i^{(t+1)}(o_k) = \frac{\sum_{t=1}^{T}\gamma_t(i)\,\mathbf{1}[x_t = o_k]}{\sum_{t=1}^{T}\gamma_t(i)}}$$

**Interpretation:** Expected number of times we are in state $q_i$ and observe $o_k$, divided by expected number of times we are in state $q_i$.

## 7. Decoding: Viterbi Algorithm

The Viterbi algorithm finds the **single most likely state sequence**:

$$\hat{I} = \arg\max_{I} P(I \mid O, \lambda) = \arg\max_I P(O, I \mid \lambda)$$

### 7.1 Viterbi Variable

Define $\delta_t(i)$ as the probability of the most likely partial path ending in state $q_i$ at time $t$:

$$\delta_t(i) = \max_{z_1,\ldots,z_{t-1}} P(x_1,\ldots,x_t,\; z_1,\ldots,z_{t-1},\; z_t=q_i \mid \lambda)$$

**Initialisation:**
$$\delta_1(i) = \pi_i\, b_i(x_1), \qquad \psi_1(i) = 0$$

**Recursion:**

$$\delta_{t+1}(j) = \max_{1\leq i\leq N}\;\left[\delta_t(i)\,a_{ij}\right]\cdot b_j(x_{t+1})$$

Store the back-pointer for path recovery:

$$\psi_{t+1}(j) = \arg\max_{1\leq i\leq N}\;\delta_t(i)\,a_{ij}$$

**Termination:**

$$P^* = \max_{1\leq i\leq N}\delta_T(i), \qquad \hat{z}_T = \arg\max_{1\leq i\leq N}\delta_T(i)$$

**Back-tracking:** Recover the optimal path by following back-pointers:

$$\hat{z}_t = \psi_{t+1}(\hat{z}_{t+1}), \qquad t = T-1,\; T-2,\;\ldots,\; 1$$

## 8. Filtering, Smoothing, and Prediction

These three tasks differ in which observations are used to estimate the hidden state.

### 8.1 Filtering (Online)

**Task:** Estimate the current state given all observations **up to now**.

$$P(z_t \mid x_{1:t},\lambda) \propto P(z_t, x_{1:t} \mid \lambda) = \alpha_t(z_t)$$

Filtering is proportional to the forward variable $\alpha_t$. It can be computed **online** (causally) since it only requires past observations.

### 8.2 Smoothing (Offline)

**Task:** Estimate the state at time $t$ given the **full** observation sequence $x_{1:T}$ (requires seeing the future).

$$P(z_t \mid x_{1:T},\lambda) = \frac{P(x_{1:T}, z_t \mid \lambda)}{P(x_{1:T} \mid \lambda)}$$

Factor the joint using the conditional independence of past and future given $z_t$:

$$P(x_{1:T}, z_t) = P(x_{1:t}, z_t)\cdot P(x_{t+1:T} \mid z_t) = \alpha_t(z_t)\cdot\beta_t(z_t)$$

Therefore:

$$\boxed{P(z_t \mid x_{1:T},\lambda) \propto \alpha_t(z_t)\,\beta_t(z_t) = \gamma_t(z_t)}$$

Smoothing requires the full forward-backward pass and produces more accurate estimates than filtering.

### 8.3 Prediction

**Task:** Predict the next hidden state given observations so far.

$$P(z_{t+1} \mid x_{1:t},\lambda) = \sum_{z_t} P(z_{t+1} \mid z_t)\, P(z_t \mid x_{1:t},\lambda)$$

This is a **Chapman-Kolmogorov** equation: propagate the filtered distribution forward one step using the transition matrix.

**Predicting the next observation:**

$$P(x_{t+1} \mid x_{1:t},\lambda) = \sum_{z_{t+1}} P(x_{t+1} \mid z_{t+1})\, P(z_{t+1} \mid x_{1:t},\lambda)$$

Average the emission probability over the predicted state distribution.

## 9. Summary

```
HMM = λ = (π, A, B)
    π: initial state distribution
    A: N×N transition matrix      P(z_{t+1}|z_t)
    B: N×M emission matrix        P(x_t|z_t)

Key algorithms:
  Forward:      α_{t+1}(j) = b_j(x_{t+1}) Σ_i α_t(i) a_{ij}
  Backward:     β_t(i)     = Σ_j a_{ij} b_j(x_{t+1}) β_{t+1}(j)

  Filtering:    P(z_t|x_{1:t})  ∝ α_t        [online]
  Smoothing:    P(z_t|x_{1:T})  ∝ α_t · β_t   [offline]
  Prediction:   P(z_{t+1}|x_{1:t}) = A^T · P(z_t|x_{1:t})

  Viterbi:      δ_{t+1}(j) = max_i [δ_t(i) a_{ij}] · b_j(x_{t+1})
  Baum-Welch:   EM with γ_t and ξ_t as sufficient statistics
```
